Glossary

  • Basically a project’s folder. A repository contains all the project files (including documentations) and stores each file’s revision history. They can private or public.
  • Fetch: Getting the latest changes from an online repository.
  • Fork: A personal copy of another user’s repository.
  • Pull: Fetching in changes and merging them. If someone has edited the remote file you’re both working on, you’ll want to pull in those changes to your local copy so it is up to date.
  • Commit: A revision of an individual file (or a set of files)
  • Push: Sending committed changes to a remote repository. Therefore other people can access them.
  • Diff: Visual description of the difference in changes between two commits.
  • Merge: Takes changes from one branch and applies them into another.
